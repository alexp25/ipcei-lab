## 📌 Context și Motivație

FRDM-MCXA153 include senzorul digital de temperatură **NXP P3T1755DP**, referință **U8**. Senzorul este conectat la interfața I3C0 a plăcii, dar poate comunica și în modul compatibil I2C cu două fire. În acest laborator folosim controlerul **LPI2C0** și API-ul MCUXpresso SDK.

Manualul plăcii poate indica adresa `0x90`. Aceasta este adresa I2C de **8 biți pentru write**. API-urile normale LPI2C primesc adresa de **7 biți**:

```text
0x90 >> 1 = 0x48
```

> **Board:** FRDM-MCXA153 · MCX A153 (Cortex-M33) · MCUXpresso SDK · VS Code + CMake

## 🎯 Obiective

3. Separarea driverului P3T1755 de adaptorul hardware și de logica aplicației.

5. Afișarea temperaturii în grade Celsius la fiecare `500 ms`.
6. Înțelegerea relației dintre `.mex`, fișierele generate și fișierele compilate de CMake.

## 🔌 Conexiunea Hardware

Senzorul on-board folosește următorii pini:

| Semnal | Pin MCU | Funcție mux |
|---|---|---|
| SDA | `P0_16` | `LPI2C0_SDA`, `Alt2` |
| SCL | `P0_17` | `LPI2C0_SCL`, `Alt2` |

Caracteristicile electrice folosite în proiect sunt:

- input buffer activ;
- open-drain activ;
- pull-up activ;
- slew rate fast;
- drive strength low.

Activarea pull-up-urilor a fost importantă la bring-up. Fără o stare idle HIGH stabilă, LPI2C0 poate raporta magistrala ca fiind ocupată înainte de primul transfer.

## 🗺️ Harta Registrelor

| Registru | Adresă | Rol |
|---|---:|---|
| Temperature | `0x00` | valoarea temperaturii, 2 octeți |
| Configuration | `0x01` | configurarea senzorului |
| TLOW | `0x02` | prag inferior |
| THIGH | `0x03` | prag superior |

Inițializarea driverului citește registrul `Configuration` fără să modifice setările power-on. Astfel verifică prezența senzorului înainte de bucla principală.

## 📡 Tranzacția I2C

Pentru citirea temperaturii:

```text
Adresă dispozitiv: 0x48, 7 biți
Write: [0x00]       selectează registrul Temperature
Repeated START
Read:  [MSB, LSB]
STOP
```

În implementare, `lpi2c_master_transfer_t` descrie aceeași operație:

```c
transfer.slaveAddress = 0x48U;
transfer.direction = kLPI2C_Read;
transfer.subaddress = P3T1755_REG_TEMPERATURE;
transfer.subaddressSize = 1U;
transfer.data = data;
transfer.dataSize = 2U;
transfer.flags = kLPI2C_TransferDefaultFlag;
```

Transferul este pornit cu `LPI2C_MasterTransferNonBlocking()`. Un callback salvează rezultatul, iar o limită software oprește așteptarea și apelează `LPI2C_MasterTransferAbort()` dacă transferul nu se termină.

## 🌡️ Conversia Temperaturii

Temperatura este un număr semnat pe 12 biți, în complement față de 2, aliniat la stânga în cuvântul de 16 biți. Rezoluția este:

```text
0.0625 °C / LSB
```

```c
int16_t p3t1755_decode_raw(uint8_t msb, uint8_t lsb)
{
    int16_t raw = (int16_t)((((uint16_t)msb << 8U) |
                             (uint16_t)lsb) >> 4U);

    if ((raw & 0x0800) != 0)
    {
        raw |= (int16_t)0xF000;
    }

    return raw;
}

float p3t1755_raw_to_celsius(uint8_t msb, uint8_t lsb)
{
    return (float)p3t1755_decode_raw(msb, lsb) * 0.0625F;
}
```

Valori de verificare:

| Raw pe 12 biți | Valoare semnată | Temperatură |
|---:|---:|---:|
| `0x190` | `400` | `25.0 °C` |
| `0x000` | `0` | `0.0 °C` |
| `0xE70` | `-400` | `-25.0 °C` |

> Shift-ul este `4`, nu `5`. P3T1755 folosește 12 biți și `0.0625 °C/LSB`, nu formatul mai vechi de 11 biți cu `0.125 °C/LSB` întâlnit la alți senzori.

## 🧱 Organizarea Codului

Implementarea este separată în trei niveluri:

```text
main.c
   │  afișare și perioadă de 500 ms
   ▼
onboard_temperature.c/.h
   │  clock, LPI2C0, callback, timeout
   ▼
p3t1755.c/.h
      registre, API senzor, conversie temperatură
```

### Driverul reutilizabil `p3t1755`

Driverul nu include apeluri directe la LPI2C0. Primește funcții de transport prin configurație:

```c
typedef struct
{
    void *busContext;
    p3t1755_read_transfer_t readTransfer;
    p3t1755_write_transfer_t writeTransfer;
    uint8_t deviceAddress;
    uint32_t timeout;
} p3t1755_config_t;
```

API-ul public include:

```c
status_t p3t1755_init(...);
status_t p3t1755_read_register(...);
status_t p3t1755_write_register(...);
status_t p3t1755_read_raw(...);
status_t p3t1755_read_celsius(...);
```

Această separare permite reutilizarea driverului cu alt controler I2C sau cu un adaptor I3C fără schimbarea logicii de conversie.

### Adaptorul hardware `onboard_temperature`

Adaptorul de placă deține LPI2C0 și pregătește clock-ul înainte de inițializare:

```c
CLOCK_SetClockDiv(kCLOCK_DivLPI2C0, 1U);
CLOCK_AttachClk(kFRO_HF_DIV_to_LPI2C0);

LPI2C_MasterGetDefaultConfig(&masterConfig);
masterConfig.baudRate_Hz = 100000U;
LPI2C_MasterInit(LPI2C0, &masterConfig, CLOCK_GetLpi2cClkFreq());
```

Clock-ul este inițializat local de modul deoarece LPI2C0 nu este instanțiat în unealta **Peripherals**. Config Tools poate regenera `clock_config.c` fără să elimine funcționalitatea aplicației.

### Integrarea în `main.c`

```c
status_t status = onboard_temperature_init();

while (1)
{
    if (status == kStatus_Success)
    {
        float temperatureCelsius;
        status_t readStatus =
            onboard_temperature_read_celsius(&temperatureCelsius);

        if (readStatus == kStatus_Success)
        {
            PRINTF("P3T1755 temperature: %.2f C\r\n",
                   (double)temperatureCelsius);
        }
        else
        {
            PRINTF("P3T1755 read failed (status %d)\r\n",
                   (int)readStatus);
        }
    }

    SDK_DelayAtLeastUs(500000U, SystemCoreClock);
}
```

Aplicația nu conține apeluri LPI2C specifice platformei și nu folosește LED-ul sau butonul. Exemplul poate fi dezactivat ușor prin eliminarea inițializării și citirii din `main.c`.

## 🛠️ Configurarea în MCUXpresso Config Tools

Deschideți fișierul:

```text
src/lab_i2c/main/main.mex
```

În **Pins**, funcția `BOARD_InitPins` trebuie să conțină:

- `LPI2C0_SDA` pe `P0_16`;
- `LPI2C0_SCL` pe `P0_17`;
- open-drain și pull-up active pentru ambele semnale.

Fișierele generate sunt grupate în:

```text
frdmmcxa153/board/pin_mux.c
frdmmcxa153/board/pin_mux.h
frdmmcxa153/board/clock_config.c
frdmmcxa153/board/clock_config.h
frdmmcxa153/board/peripherals.c
frdmmcxa153/board/peripherals.h
```

CMake trebuie să compileze exact aceste fișiere. Dacă dialogul **Update Files** propune `create` într-un director și `delete` în altul, verificați `Project Link`, `generated_project_files`, `board_files.cmake` și `cfg_tools_generated.cmake` înainte de a apăsa **OK**.

Pentru explicația generală despre `.mex`, ownership și alinierea cu build-ul, consultați secțiunea dedicată din [L1 — GPIO](../l1-gpio#cum-se-leagă-mex-config-tools-codul-generat-și-build-ul).

## 🔍 Bring-up și Diagnosticare

În MCUXpresso SDK, codurile `status_t` includ grupul modulului și codul erorii. În timpul integrării au fost întâlnite următoarele situații:

| Simptom | Interpretare | Corecție |
|---|---|---|
| `status 7911` | timeout pe încercarea inițială I3C legacy-I2C | verificarea modului de controler și a tranzacției |
| `status 7902` | NAK în varianta I3C cu dynamic address assignment | folosirea modului compatibil LPI2C la adresa `0x48` |
| break în `fsl_assert` | clock-ul LPI2C0/divizorul nu era pregătit | setarea divizorului și atașarea sursei înainte de `LPI2C_MasterInit()` |
| `status 900` | `kStatus_LPI2C_Busy`; magistrala nu era idle | configurarea corectă a pinilor și activarea pull-up-urilor |
| Config Tools șterge linii de clock | clock-ul era modificat numai în fișierul generat | mutarea ownership-ului în adaptorul hardware sau configurarea completă în unealtă |
| `Invalid settings` în Clocks | identificator XML adăugat manual, neacceptat de model | folosirea numai a setărilor expuse de Config Tools |

### Ordinea recomandată de verificare

1. Confirmați alimentarea plăcii și adresa `0x48`.
2. Verificați în Config Tools `P0_16/P0_17`, `Alt2`, open-drain și pull-up.
3. Confirmați că CMake compilează fișierele generate din `frdmmcxa153/board`.
4. Verificați clock-ul înainte de `LPI2C_MasterInit()`.
5. Reduceți temporar magistrala la `100 kHz` pentru bring-up.
6. Verificați dacă transferul pornește și dacă callback-ul se execută.
7. Păstrați timeout-ul; nu așteptați infinit după un senzor absent.
8. După orice **Update Code**, executați un clean build și retestați placa.

## ⚠️ Capcane Critice

- `0x90` este adresa write pe 8 biți; pentru LPI2C folosiți `0x48`.
- Pinii senzorului sunt `P0_16/P0_17`, nu `PIO0_12/PIO0_13`.
- Senzorul este U8 și folosește format semnat pe 12 biți cu `0.0625 °C/LSB`.
- Open-drain fără pull-up poate lăsa magistrala în starea `Busy`.
- Nu modificați permanent numai fișiere generate; următorul **Update Code** le poate suprascrie.
- Nu introduceți manual setări de clock necunoscute în XML-ul `.mex`.
- Driverul de senzor nu trebuie să depindă direct de `LPI2C0` sau de `main.c`.
