## 📌 Context și Motivație

P3T1755DPJ este un senzor de temperatură I3C/I2C **on-board** pe FRDM-MCXA153 (U5, lângă conectorul Mikroe). Folosit prin I2C standard (../lPI2C0) la adresa `0x48`. Rezoluție: 11 biți semnificativi, format: 2 octeți complementul față de 2 (MSB first). **Nu e nevoie de componente externe.** Relevanță directă: thermal management în sisteme automotive (senzori temperatură baterie EV).

> **Board:** FRDM-MCXA153 · MCX A153 (Cortex-M33 @ 96 MHz) · SDK MCUXpresso 24.12 · VS Code + CMake

## 🎯 Obiective

1. LPI2C0 configurat la 400 kHz Fast Mode — comunicație cu P3T1755 la adresa 0x48
2. Funcție `float p3t1755_read_temp_celsius(void)` — citire și decodare registru temperatură
3. Termometru funcțional: temperatura pe UART la 1 Hz + LED roșu dacă temp > 30°C
4. Configurare registru TOS (Over-temperature Shutdown) la 30°C prin I2C write

### Prompt: LPI2C + P3T1755 temperatură

```text
Context hardware: FRDM-MCXA153, MCX A153 Cortex-M33 @ 96 MHz, SDK MCUXpresso 24.12.
Senzor on-board: P3T1755DPJ la adresa I2C 0x48. LPI2C0 (SDA=PIO0_12, SCL=PIO0_13).
Frecvență I2C: 400 kHz (Fast Mode).
Sarcina: float p3t1755_read_temp_celsius(void).
Format registru temperatură 0x00: 2 octeți MSB first, 11 biți efectivi.
Formula: temp_raw = (int16_t)(byte1 << 8 | byte2) >> 5;
         temp_c = temp_raw * 0.125f;
Include:
  LPI2C_MasterGetDefaultConfig(), LPI2C_MasterInit()
  LPI2C_MasterStart(), LPI2C_MasterWriteBlocking()
  LPI2C_MasterRepeatedStart(), LPI2C_MasterReadBlocking()
  LPI2C_MasterStop()
```

### Prompt: TOS alert threshold

```text
FRDM-MCXA153, P3T1755 la 0x48, LPI2C0 funcțional.
Sarcina: void p3t1755_set_threshold(float temp_c) — scrie registrul TOS (0x03).
Format TOS: identic cu registrul de temperatură (11 biți, MSB first, complementul față de 2).
Calculul pentru 30.000°C: 30 / 0.125 = 240 = 0x0F0
Shiftat stânga cu 5: 0x1E00 → MSB=0x1E, LSB=0x00.
Secvența I2C write: START → adresă+W → 0x03 → 0x1E → 0x00 → STOP.
```

## ⚠️ Capcane Critice

> Ce GenAI **nu știe** fără context explicit — verificați înainte de upload pe placă.

- **Secvența corectă write registru**: START → adresă+W → [reg_addr, data_msb, data_lsb] → STOP (un singur transfer, nu două)
- **Format temperatură**: shift right cu 5 **după** asamblarea celor 2 octeți — greșeală frecventă de AI
- Pull-up-urile I2C sunt montate pe placă (R47, R48) — nu e nevoie de rezistoare externe la 400 kHz
